# Opening audits and roadmap (2026-07-01, session start)

Immediately after "i want to make this website world-class", two audit agents ran in parallel (~4 min each): a frontend audit (96k tokens, 47 tool uses) and a backend audit (77k tokens, 36 tool uses). Their findings shaped everything that followed. Part of [project overview](2026-07-project-overview.md).

## Frontend audit (React 19 + Vite 6 + Tailwind 4 + shadcn/ui, Lovable/GPT-Engineer scaffold)

**Already good:** MediaPlayer (blob caching, error mapping, ARIA slider, pause-others-on-play); multi-modal CreatePost; the tag/filter system; FOUC-safe theming; CSRF-aware fetch wrapper; Radix dialogs.

**Ranked gaps:**
- P0: no pagination/infinite scroll — `getPosts` fetched the entire table; a `h-96` div as a manual bottom spacer.
- P0: likes/comments/share were `console.log` stubs; comment count hardcoded 0.
- P0: Profile page 100% mock data ("1,024 Followers", San Francisco) and passed `<Post>` without the required `onEdit` prop — a type error and crash path.
- P1: all three unit-test files targeted an obsolete API contract (string ids, `{posts:[...]}` shape, `timestamp` fields) — dead suite giving false confidence.
- P1: `handleEditPost` fired the success toast without awaiting the mutation — failed edits reported success.
- P1: no optimistic updates anywhere.
- P1: feed fetch was an N+1 presign waterfall — one `/uploads/presign/{id}/` call per media post, gating first paint on the slowest.
- P1: SEO/PWA missing — Vite-logo favicon, empty meta description, `cdn.gpteng.co` script, `lovable-tagger` plugin, no manifest.
- P2: no code splitting (everything in one bundle incl. unused recharts/embla/sidebar), no ErrorBoundary, `credentials: 'include'` missing, skeletons unused, a11y gaps (missing alt/aria-labels), double toast systems, zero-config QueryClient.

## Backend audit (Django 5.2 + DRF, SQLite, Cloudflare R2, Whisper)

**Already good:** author/admin authorization on edit/delete (tested), real CSP with per-script hashes, thorough file cleanup on delete, 16-char password minimum, secrets from env.

**Security-critical:**
- S1: migration `users/0002` created superuser `admin`/`admin` on every fresh DB — and `fly.toml` runs migrate on every deploy. Full admin takeover.
- S2: presign endpoint unauthenticated with attacker-controlled key/content-type, no limits, `public-read` ACL — an open world-writable object store + key traversal.
- S3: transcribe endpoint unauthenticated — anyone could burn OpenAI credits.
- S4: `get_presigned_url_for_post` leaked any post's media, no auth.
- S5–S8: no throttling anywhere, zero transport hardening (no SECURE_PROXY_SSL_HEADER/secure cookies/HSTS), password validators disabled under DEBUG.

**Correctness:**
- C1 (the big one): every media path called `.file.path`, which raises `NotImplementedError` under `S3Boto3Storage` — the whole media pipeline (duration, mp3, streaming, transcription, deletion) only worked on local disk; prod R2 config was broken.
- C2/C3: plain Django views returned DRF `Response` objects (guaranteed 500s); create-response serialized without request context.
- C5: range-request handling was non-functional (bogus `HTTPS_RANGE` header on secure requests, `== 2` magic check, indexing the werkzeug Range object itself).
- C7/C8: `convert_to_mp3` swallowed ffmpeg failures; transcription leaked temp files and had no API timeout.
- C9/C10: hardcoded anonymous user id 2; Media PK force-set to Post PK.

**Perf/ops:** list N+1 (no prefetch), no pagination, everything synchronous in-request (no job queue), SQLite in prod, no error tracking, no health check, CI never ran tests (deploy-only workflow).

## The roadmap

Proposed: Phase 0 security → Phase 1 correctness → Phase 2 social → Phase 3 polish. Asked Tam where to start via AskUserQuestion; **Tam chose "Social features first"** over the recommended security phase, so execution order became 2 → 0 → 1 → review → 3. Every audit finding above was eventually addressed or explicitly deferred with rationale (see the phase reports).
